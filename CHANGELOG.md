# Changelog

### 2026-05-26
- В `api/openapi.yaml` изменен блок `info`: добавлен/перенесен `title` со значением `API Яндекс Маркета для продавцов`.
- Описание API в `info.description` сокращено до одного абзаца про автоматизацию и упрощение работы с маркетплейсом.
- Из спецификации удален прежний заголовок `Партнерский API Маркета`, версия `LATEST`, лицензия BSD 3-Clause и ссылка `externalDocs`.
- Из видимого фрагмента удалены глобальные `servers` и `security`.
- По shortstat видно крупное сокращение спецификации: `339` добавлений и `45812` удалений в одном файле.
- В удаленном контексте видны удаляемые описания путей, включая `/businesses/{businessId}/settings`, `/campaigns`, `/campaigns/{campaignId}`, `/campaigns/{campaignId}/settings`, `/campaigns/{campaignId}/region`, `/campaigns/{campaignId}/orders/{orderId}` и `/campaigns/{campaignId}/orders`.
- Детали оставшихся `339` добавлений по предоставленному фрагменту недостаточны: видны только изменения начала файла и большой удаленный блок.
- Риски: изменения потенциально несовместимы с предыдущей версией SDK, так как из видимого diff удаляются глобальные настройки и множество endpoint-описаний; точный масштаб breaking changes нельзя подтвердить без полного diff.
### 2026-05-28
- В `ClosureDocumentsContractType` из описания удалено значение `MARKETING` как договор на маркетинг.
- В `GenerateClosureDocumentsDetalizationRequest` поле `campaignId` добавлено в список обязательных.
- В `openapi.yaml` изменено оформление метаданных `info`: добавлены/переставлены `title`, `description` и `version: LATEST`.
- В `openapi.yaml` секции `servers` и `security` вынесены выше, сразу после `info`.
- Большие inline-описания путей в `openapi.yaml` заменены на `$ref` к файлам `paths/*.yaml`, например для `v2_businesses_businessId_settings`, `v2_campaigns` и заказов.
- Изменения затрагивают 4 файла; большая часть diff выглядит как структурная декомпозиция спецификации, но полный список затронутых endpoint'ов по предоставленному фрагменту определить нельзя.
- Риски: добавление обязательного `campaignId` может быть обратно несовместимым для клиентов генерации документов детализации; остальное по видимому контексту в основном похоже на реорганизацию OpenAPI-файла и правки документации.
### 2026-06-02
- В `CreateOrderDTO` добавлено поле `fake` для создания тестового заказа, значение по умолчанию — `false`.
- Описание метода создания заказа дополнено поведением параметра `fake`: тестовый заказ не отгружается и не влияет на остатки.
- В `OrderSubstatusType` добавлен новый подстатус `USER_IDENTIFICATION_MISMATCH`.
- В описании `OrderPaymentMethodType` изменено расположение пункта `BNPL_TBYB`; новых enum-значений по видимому контексту не добавлено.
- Для ряда методов генерации отчетов и документов добавлены read-only scope, включая `all-methods:read-only`, `promotion:read-only`, `inventory-and-order-processing:read-only`, `offers-and-cards-management:read-only` и `pricing:read-only`.
- В `v2_campaigns_campaignId_stats_orders` изменен путь в расширении с `resp.result.orders` на `resp`, также удален `tariff-path: resp`.
- Контекст недостаточен, чтобы точно перечислить все затронутые методы и оценить изменения в сгенерированных Go-типах, так как показан только фрагмент diff.
- Риски: добавление поля и enum-значения выглядит обратно совместимым, но изменение путей в `v2_campaigns_campaignId_stats_orders` и scope может повлиять на генерацию SDK или требования к авторизации.
